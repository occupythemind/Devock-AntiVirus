Welcome to the Devock documentation! Here you will find all the information you need to get started with Devock and make the most of its features.

Malware attacks on devices increase per day especially on devices owned by big industries as in the case of ransomwares. After studying the nature of these attacks, I realized that 60-70% of these attacks are caused by the owners/handlers of these devices, either by downloading a file without checking it out first. Although I wouldn't blame all cause not everyone understands this things. Now, this is where Devock comes in; Devock Antivirus is a project that helps prevent these malicious software from infecting a device. Although, this version is specific to Linux but we're cooking up something for windows and Mac OS, it will be released as soon as it's done.



What Devock Does

Devock operates at the network level, using a python module mitmproxy which allows it to listen and intercept traffic going and coming into a device. Devock listens for response from servers, checks if the response is an application/file that is being downloaded then intercepts the traffic and copies the bytes of the file. Once copied, it writes the bytes to a file and quarantines that file in the directory /mnt/quarantine1. After quarantine, it scans the file using clamscan (Clam Antivirus--Which is a malware scanner) which would check for known malware signatures. If any malware signatures were found, it returns FOUND else OK. If the file is safe, it would be moved to the user's Download Folder else it would be deleted at unmount (I'm getting to what i mean here.) The directory /mnt/quarantine1 acts as a mount point for tmpfs -- which is a temporary storage. When it's unmounted, all data incuding the file will be wiped off, which is an advantage.



How to start

On downloading Devock, a script comes with it named install.sh Run the script using bash

~$ sh install.sh or, you make it executable
~$ chmod +x install.sh
~$ ./install.sh

Configure your browser proxy to 127.0.0.1:8080 which is the default proxy for mitmproxy.
Start up mitmproxy

~$ mitmproxy or
~$ mitmweb

Head to mitm.it to download a certificate (In this case, you'll be downloading for linux) which should end with .pem. Follow the instructions for the certificate. If you're downloading for firefox, you'll have to download the required certificate and also follow the instructions from the site. With this certificate, mitmproxy will be able to decrypt traffic (HTTPS) inorder to analyze it's content.

After you've run install.sh, a systemd service file should be created. This should allow Devock run in the background even on reboot. But incase of errors, you can run Devock manually using the autorun.sh file in the same folder as the install.sh file.

~$ sh autorun.sh or
~$ ./autorun.sh (If the script is executable)
Now, as you browse, Devock will constantly check the files you download before allowing them in.
